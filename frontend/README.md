# VillageVista — Frontend

A lightweight React + Vite frontend for VillageVista: an AI travel assistant focused on rural, eco, and off‑beat stays. The UI lets users ask natural-language questions and receives destination cards generated by Mistral AI and enriched with real-time web context from Tavily.

**Tech Stack**
- **Framework:** `React` (with Vite)
- **State:** `zustand`
- **Styling:** `Tailwind CSS`
- **AI & Web Search:** Mistral (chat completions) + Tavily (search enrichment)
- **Tooling:** `vite`, `eslint`, optional `bun` or `npm` for running

**Key Features**
- Natural-language travel chat assistant (floating chat UI)
- Destination cards returned as structured JSON by the AI and rendered as cards
- Real-time web context via Tavily to improve factuality
- Wikimedia/Unsplash fallback image fetching for destination photos
- Recent searches, suggestions, and smooth scroll-to-results behavior

**How it works (overview)**
- User message → `handleUserQuery()` ([src/utils/api.js](src/utils/api.js))
- If the query looks like a destination request (see `shouldUseTavily()`), the app:
  1. Calls Tavily search (`searchWithTavily`) to gather web context
  2. Builds a system + user prompt (`buildDestinationPrompt`) including web context
  3. Sends messages to Mistral via `askMistral()` and receives a text reply
  4. Parses a JSON array inside <destinations>...</destinations> (`parseDestinationsFromAI`)
  5. Fetches images (`fetchDestinationImages`) and attaches them to cards
  6. Writes destinations to the store for the UI to render

**Important files**
- [src/utils/api.js](src/utils/api.js) — orchestration, prompts, Mistral/Tavily calls
- [src/hooks/useChatQuery.js](src/hooks/useChatQuery.js) — hook used by the UI to send queries

Running locally

1. Clone the repo and enter the frontend folder:

```
git clone <repo-url>
cd frontend
```

2. Install dependencies (npm) or with bun:

```
npm install
# or
bun install
```

3. Create a `.env` file in `frontend/` (see .env.example below) and add your keys.

4. Start the dev server:

```
npm run dev
# or with bun
bun run dev
```

Build and preview

```
npm run build
npm run preview
```

Linting

```
npm run lint
```

.env and required API keys

Create a `.env` file at the project root of the frontend (do NOT commit it). The app expects these variables (Vite uses the `VITE_` prefix):

- `VITE_MISTRAL_API_KEY` — Bearer API key for Mistral (chat completions)
- `VITE_TAVILY_API_KEY` — API key for Tavily search
- `VITE_MISTRAL_BASE_URL` — optional, defaults to `https://api.mistral.ai/v1/chat/completions`
- `VITE_TAVILY_BASE_URL` — optional, defaults to `https://api.tavily.com/search`
- `VITE_MISTRAL_MODEL` — optional, defaults to `mistral-small-latest`

Example `.env` (replace placeholders with your own keys):

```
VITE_MISTRAL_API_KEY=your_mistral_api_key_here
VITE_TAVILY_API_KEY=your_tavily_api_key_here
VITE_MISTRAL_BASE_URL=https://api.mistral.ai/v1/chat/completions
VITE_TAVILY_BASE_URL=https://api.tavily.com/search
VITE_MISTRAL_MODEL=mistral-small-latest
```

Security note: Never commit real API keys to source control. Add `frontend/.env` to `.gitignore`.

Endpoints & models used

- Mistral chat completions endpoint (default): `https://api.mistral.ai/v1/chat/completions`
- Tavily search endpoint (default): `https://api.tavily.com/search`

These endpoints are used by `askMistral()` and `searchWithTavily()` in [src/utils/api.js](src/utils/api.js).

Default Mistral model

- The app uses `VITE_MISTRAL_MODEL` and defaults to `mistral-small-latest` (configured in [src/utils/api.js](src/utils/api.js)).
- You can swap other Mistral models if available for your account (for example: `mistral-large`, `mistral-instruct`, or other hosted variants) — changing `VITE_MISTRAL_MODEL` and restarting the dev server is sufficient.

Getting API keys

**Mistral (API key)**

1. Open https://studio.mistral.ai/ and sign up or sign in with your account.
2. In the Studio dashboard, go to your account or workspace settings (often under a profile or team menu).
3. Look for "API keys" or "Tokens" and click "Create" or "Generate" to make a new key.
4. Copy the key immediately and store it securely (private vault or environment manager).
5. Add the key to your `frontend/.env` as `VITE_MISTRAL_API_KEY`.

Notes:
- Some Mistral plans or organizations may restrict model access — ensure your account has access to the model you want (for example `mistral-small-latest` or `mistral-large`).
- If your organization issues scoped keys, prefer a key limited to chat completions for the frontend.

**Tavily (API key)**

1. Visit https://tavily.com/ and sign up for a developer or business account (follow the site's signup flow).
2. After logging in, open the developer dashboard or API section where keys or tokens are managed.
3. Create a new API key for your project; name it (e.g., "VillageVista frontend dev").
4. Copy the API key and store it securely.
5. Add the key to your `frontend/.env` as `VITE_TAVILY_API_KEY`.

Notes:
- Tavily may provide usage limits or billing—check their dashboard for quotas and pricing.
- If Tavily supports IP restrictions or referer restrictions, configure them for your deployment origin.

Security best practices

- Never commit API keys to git. Add `frontend/.env` to `.gitignore`.
- Use separate keys for development and production with limited scopes when possible.
- Rotate keys periodically and revoke keys that are no longer in use.

Where to change behavior

- Prompt templates and parsing rules: [src/utils/api.js](src/utils/api.js)
- The chat UI and input flow: [src/components/ChatAssistant.jsx](src/components/ChatAssistant.jsx)

Troubleshooting

- If AI requests fail, check `VITE_MISTRAL_API_KEY` and `VITE_MISTRAL_BASE_URL`.
- If web context is missing, check `VITE_TAVILY_API_KEY` and `VITE_TAVILY_BASE_URL`.
- Network errors will appear in the browser console; server-side HTTP errors are surfaced by exceptions in `src/utils/api.js`.
